# 무료 호스팅 가이드

## ⭐️ 추천 1: Vercel (프론트) + Render (백엔드) - 완전 무료!

### Vercel로 프론트엔드 배포
1. https://vercel.com 접속 → GitHub로 가입
2. "Add New Project" 클릭
3. GitHub 레포지토리 선택
4. 자동으로 감지하여 Next.js 설정 (빌드 설정 자동)
5. "Deploy" 클릭 → 자동 배포 완료!

**무료 플랜 특징:**
- ✅ 완전 무료 (Hobby 플랜)
- ✅ 무제한 배포
- ✅ 자동 HTTPS
- ✅ CDN 자동 적용
- ✅ GitHub 연동 자동 배포

### Render로 백엔드 배포
1. https://render.com 접속 → GitHub로 가입
2. "New Web Service" 선택
3. GitHub 레포지토리 연결
4. 설정:
   - **Root Directory**: `backend`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python main.py`
   - **Environment**: Python 3
   - **Plan**: Free (무료)

**장점:**
- ✅ 완전 무료 (프론트 + 백엔드 모두)
- ✅ Vercel은 Next.js에 최적화
- ✅ 각각 독립적으로 스케일링 가능

**주의사항:**
- ⚠️ Render 무료 티어는 15분 비활성 시 슬립
- ⚠️ 슬립 후 첫 요청 시 깨어나는 시간 필요 (약 30초)

## 📝 공통 주의사항

### CORS 설정 변경 필요
배포 후 백엔드 `main.py`의 CORS 설정을 배포된 프론트엔드 URL로 변경해야 합니다:

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:3000",
        "https://your-frontend-domain.com"  # 배포된 도메인 추가
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 환경 변수
다음 환경 변수를 설정해야 할 수 있습니다:
- OpenAI API Key
- Groq API Key
- Gemini API Key
- Notion API Keys

- 트래픽 제한
- 리소스 제한 (CPU, 메모리)
- 슬립 모드 (Render의 경우)
