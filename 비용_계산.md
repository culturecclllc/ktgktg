# 호스팅 비용 계산 (11명 사용 기준)

## 📊 예상 사용량

**사용자 수:** 11명  
**하루 1인당:**
- 초안 요청: 3-4개
- 최종글 요청: 3-4개

**하루 전체:**
- 초안 요청: 약 33-44개 (평균 38개)
- 최종글 요청: 약 33-44개 (평균 38개)
- **총 API 요청: 약 76개/일**

**월간 전체:**
- 초안 요청: 약 1,140개
- 최종글 요청: 약 1,140개
- **총 API 요청: 약 2,280개/월**

**실제 백엔드 요청 (상세):**
- 초안 1개당: 3개 모델 = 3번 API 호출
- 최종글 1개당: 장단점 분석 3개 모델 + 최종 생성 1번 = 4번 API 호출
- **월간 백엔드 요청: 약 1,140 × 3 + 1,140 × 4 = 약 7,980회**

## 💰 무료 플랜 한도 분석

### 1. Vercel (프론트엔드) - ✅ 충분

**무료 플랜(Hobby) 한도:**
- Function Invocations: 월 100만 회
- Bandwidth: 월 100GB
- 빌드 시간: 월 6000분

**예상 사용량:**
- 프론트엔드 요청: 월 약 2,280회
- Bandwidth: 소규모

**결론:** ✅ **완전 무료로 충분합니다!**

---

### 2. Render (백엔드) - ⚠️ 주의 필요

**무료 플랜 한도:**
- 750시간/월 (계속 실행 가능)
- 15분 비활성 시 슬립 모드

**예상 사용량:**
- 백엔드 실행 시간: 24시간 × 30일 = 720시간
- **11명 사용하면 계속 활성 상태 유지 가능**

**결론:** 
- ✅ **무료 플랜으로 가능하지만**
- ⚠️ **슬립 모드 문제**: 비활성 시 30초 지연 발생
- ⚠️ **추천**: Free 티어 사용 가능하지만, 사용자가 많으면 `Starter` 플랜($7/월) 고려

---

### 3. Railway (백엔드) - 💸 비용 발생 가능

**무료 플랜:**
- $5/월 크레딧 무료 제공

**예상 사용량:**
- 소규모 인스턴스: 약 $5-10/월 예상
- **무료 크레딧으로는 부족할 수 있음**

**결론:** 
- ⚠️ **초과 가능성 높음**
- 💸 **월 $0-5 정도 비용 발생 가능**

---

### 4. Fly.io (백엔드) - ✅ 가능

**무료 플랜:**
- 3개 공유 CPU VM (256MB RAM)
- 월 160GB 네트워크 전송

**예상 사용량:**
- API 요청: 월 약 7,980회
- 네트워크: 소규모

**결론:** ✅ **무료 플랜으로 충분합니다!**

## 🎯 최종 추천 (비용 기준)

### 옵션 1: Vercel + Render (Free) - ⚠️ 슬립 모드 주의
- **비용:** $0/월
- **장점:** 완전 무료
- **단점:** Render 슬립 모드로 첫 요청 지연 (약 30초)
- **적합:** 비용 절감이 최우선인 경우

### 옵션 2: Vercel + Fly.io - ⭐️ **최고 추천!**
- **비용:** $0/월
- **장점:** 완전 무료 + 슬립 모드 없음
- **단점:** 설정이 약간 복잡할 수 있음
- **적합:** 안정적인 무료 서비스 원하는 경우

### 옵션 3: Vercel + Render (Starter) - 안정적
- **비용:** $7/월 (Render만)
- **장점:** 슬립 모드 없음, 안정적
- **단점:** 월 $7 비용 발생
- **적합:** 비용 부담 없고 안정성 원하는 경우

### 옵션 4: Railway (풀스택)
- **비용:** $0-5/월 (크레딧 초과 시)
- **장점:** 한 곳에서 관리, 간편함
- **단점:** 비용 발생 가능성
- **적합:** 간편함을 원하는 경우
